土地資料系統 完整版變更紀錄
================================

一、專案起點
------------------------------
1. 一開始需求是建立一套使用 SQLite 的客戶資料系統。
2. 後續需求逐步轉向土地資料管理，因此 UI、欄位命名、資料邏輯與匯入格式都持續調整。

二、早期資料結構與欄位調整
------------------------------
1. 原本客戶資料系統改為 SQLite 土地資料系統。
2. 移除 Email 欄位。
3. 原本的公司欄位改為 ID 欄位使用。
4. 資料欄位曾依需求多次調整，主要演進如下：

第一階段：
- 區
- 段
- 地號
- 面積
- 公告現值
- 權利範圍
- 總計公告現值
- 所有權人
- ID
- 地址
- 登記原因
- 登記次序
- 備註

第二階段：
- 指定登記次序移到地段後面。

第三階段：
- UI 參考新版表頭樣式，改為：
  地區、地段、序號、地號、面積/m2、公告現值、分子、分母、坪數、
  總現值/元、姓名、身分證、地址、原因、備註、出訪記錄。

5. 為了兼容舊資料與舊匯入格式，系統仍保留舊欄位對應與資料遷移處理。

三、UI 版面與互動調整歷程
------------------------------
1. 系統從純資料庫需求改為桌面 UI 介面。
2. 使用者曾要求欄位顯示方式做多次調整：
- 曾要求上方橫向欄位顯示。
- 後續又要求右側 UI 改回一開始樣式。
- 最後採用左側表格、右側表單輸入區的工作介面。

3. 捲動與視覺需求：
- 一度要求不要可捲動的右側 UI。
- 最後保留穩定的右側表單布局。

4. 新增右鍵選單：
- 可在表格右鍵直接刪除資料。

5. 新增勾選方格：
- 可勾選多筆資料後批次清除。

6. 新增全部清除：
- 需要 3 次確認視窗後才會清空全部資料。

四、自動計算與格式化調整
------------------------------
1. 總公告現值改成可用公式自動計算。
2. 總公告現值不顯示小數點。
3. 公告現值與總公告現值加入千分位格式。
4. 坪數改為顯示到小數點後兩位。
5. 在新欄位版本中：
- 若沒有坪數，系統可由面積/m2 自動換算。
- 若沒有總現值/元，系統可由面積、公告現值、分子、分母自動換算。

五、Excel 匯入功能與修正
------------------------------
1. 新增 `.xlsx` 匯入功能。
2. 後續使用者反映：
- 中文文字沒有正常匯入。
- 有些欄位資料沒有正常匯入。

3. 依此做了匯入邏輯加強：
- 支援中文欄位名稱辨識。
- 支援多種舊表頭與新表頭對應。
- 支援模糊對應欄位名稱。
- 支援合併儲存格取值。
- 支援空白列沿用上列資料。
- 支援權利範圍拆成分子/分母。
- 支援匯入時自動補格式與公式欄位。

4. 匯入完成後會提示：
- 匯入筆數
- 成功對應的欄位

六、登入驗證與資料加密歷程
------------------------------
1. 使用者先詢問是否可以做資料加密，之後正式要求實作。
2. 再加入登入帳號驗證。
3. 因目前為單人使用，採固定帳號：
- admin

4. 密碼規則：
- 登入密碼同時也是資料解密密碼。
- 若忘記密碼，已加密資料無法解密。

5. 技術實作方向：
- 密碼使用 salted PBKDF2 hash 儲存。
- 另以密碼衍生資料解密金鑰。
- 敏感欄位改為加密儲存。

6. 加密欄位包含：
- 姓名
- 身分證
- 地址
- 備註
- 出訪記錄
- 舊相容欄位 name

7. 新增「加密既有資料」功能：
- 可把資料庫中尚未加密的舊資料轉成加密格式。

8. 曾發生問題：
- 按「加密既有資料」顯示「已處理 0 筆資料」。
- 後續已修正為正確判斷與處理。

七、身分證顯示/隱藏功能
------------------------------
1. 使用者要求加一個按鈕，可顯示或隱藏身分證。
2. 隱藏規則：
- 保留前 3 碼
- 保留最後 1 碼
- 中間改為 `*`

3. 此功能同時作用於：
- 左側表格顯示
- 右側表單欄位

4. 並避免隱藏狀態下把遮罩內容誤存回資料庫。

八、資料儲存位置與可攜化
------------------------------
1. 使用者詢問資料儲存位置。
2. 之後需求轉向：
- 打包成 EXE
- 搬到另一台電腦也可直接開啟使用

3. 因此加入可攜路徑設計：
- 一般 Python 執行時，資料庫與資源檔從程式目錄取得。
- EXE 執行時，也會把資料庫放在程式所在資料夾。

4. 這樣做的目的：
- 複製整個資料夾到另一台 Windows 即可使用
- 若帶上 `customers.db`，可保留原資料
- 若沒有資料庫，第一次啟動會自動建立

九、從 Tkinter 過渡到 PySide6
------------------------------
1. 原先 UI 是 Tkinter 版本。
2. 使用者在筆電上匯入資料後，滾動表格時感受到明顯頓挫。
3. 原因分析方向包括：
- Tkinter Treeview 大量多欄位資料重繪效能有限
- 中文長字串與高 DPI 顯示較吃資源
- 主執行緒同時處理解密、格式化、UI 重繪

4. 使用者最後要求：
- 功能一致
- 版面盡量照現在
- 優先改善表格流暢度
- 不先亂加新功能

5. 因此後續改為 PySide6 版本。

6. PySide6 版保留的核心方向：
- 左側資料表
- 右側表單
- 上方工具列
- 原有登入、加密、匯入、刪除、身分證切換等功能

7. PySide6 版的效能改善：
- 改用 QTableView + QAbstractTableModel
- 滾動改為較順的模式
- 後續再加入分批載入
- 降低大量匯入後初次刷新負擔

十、備份功能
------------------------------
1. 使用者希望系統更安全，因此新增自動備份。
2. 備份檔位置：
- `backups\`

3. 備份策略：
- 每日啟動自動備份一次
- 保留多份自動備份

4. 在高風險動作前自動備份：
- 匯入 Excel 前
- 清除勾選資料前
- 全部清除前
- 加密既有資料前
- 修改密碼前
- 還原備份前

5. 並新增設定入口：
- 開啟備份資料夾

十一、修改密碼功能
------------------------------
1. 使用者要求加入修改密碼功能。
2. 指定要放在「設定」內。
3. 因登入密碼同時也是解密密碼，所以這個功能不能只改 hash。

4. 修改密碼流程實作：
- 先驗證目前密碼
- 先備份目前資料庫
- 用目前密碼解密既有敏感欄位
- 重新產生新密碼 hash
- 重新產生新 encryption salt
- 用新密碼衍生的新金鑰重新加密資料
- 更新資料庫中的使用者驗證資訊

5. 完成後：
- 原本資料仍可正常使用
- 之後需用新密碼登入

十二、還原備份功能
------------------------------
1. 使用者接著要求加入備份還原。
2. 因為還原會直接覆蓋目前資料庫，所以加入保護流程：
- 先讓使用者選擇備份 `.db` 檔
- 顯示確認訊息
- 先備份目前資料庫
- 再以選取備份覆蓋現行 `customers.db`
- 完成後提示使用者重新開啟系統
- 程式關閉，避免還原後還用舊連線狀態繼續操作

十三、篩選與排序功能
------------------------------
1. 使用者希望再加入：
- 條件篩選
- 欄位排序

2. 因此在上方工具列新增：
- 篩選欄位選單
- 排序欄位選單
- 排序方向選單

3. 篩選設計：
- 可選全部欄位
- 也可指定地區、地段、序號、地號、姓名、身分證、地址、原因、備註、出訪記錄等欄位

4. 排序設計：
- 可依系統 ID、地區、地段、序號、地號、面積、公告現值、分子、分母、坪數、總現值/元、姓名排序
- 支援新到舊 / 舊到新

5. 保留原本的搜尋習慣：
- 關鍵字搜尋仍可用
- 只是多了指定欄位與排序方式

十四、選取格問題修正
------------------------------
1. 用戶後續回報：
- 表格中的選取格無法正常使用。

2. 問題原因：
- 雖然表格模型已有勾選狀態，但在目前的 Qt 表格配置下，
  點擊第一欄時較偏向列選取，不一定會穩定切換 checkbox 狀態。

3. 修正方式：
- 額外補上第一欄點擊事件處理。
- 使用者點擊第一欄時，直接切換該筆資料的勾選狀態。

4. 修正結果：
- 選取格恢復正常，可穩定勾選與取消勾選。

十五、EXE 打包與正式版整理
------------------------------
1. 新增 `build_exe.bat` 打包腳本。
2. 可用 PyInstaller 產生 one-folder EXE 版本。
3. 打包時會把必要資源檔一起帶入。
4. 後續每次重要功能完成後，都有重新驗證與重打包正式 EXE。
